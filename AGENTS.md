# Mini ERP - OpenCode Guidance

## Project Structure
- Django 6.0.4 with SQLite (`db.sqlite3`)
- Single app: `erp/` with Profile model (roles: admin, vendedor, almacen)
- Virtual environment: `venv/`
- Project config: `core/` (generated by `django-admin startproject`)

## Developer Commands
```bash
# Activate venv first (REQUIRED for all commands)
source venv/bin/activate

# Run dev server
python manage.py runserver

# Migrations
python manage.py makemigrations
python manage.py migrate

# Tests
# Run all tests
python manage.py test erp

# Run specific tests
python manage.py test erp.tests.ProfileModelTest
python manage.py test erp.tests.HomeViewTest

# Run with more verbosity
python manage.py test erp --verbosity=2

# Run and stop at the first error
python manage.py test erp --failfast

# Run tests with coverage (requires coverage)
coverage run manage.py test erp
coverage report

# Create superuser
python manage.py createsuperuser

# Django shell
python manage.py shell
```

## Key Patterns
- Profile auto-created via signals (`erp/signals.py`) when User is created
- Profile linked via `OneToOneField` on User model with `related_name='profile'`
- Role choices: admin, vendedor, almacen

## URLs
- `/admin/` - Django admin
- `/accounts/login/`, `/accounts/logout/` - auth views
- `/` - erp app views

## Notes
- Always `source venv/bin/activate` before Django commands
- No lint/typecheck configured (vanilla Django)
- DEBUG=True in settings
